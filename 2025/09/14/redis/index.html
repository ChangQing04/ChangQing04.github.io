Redis 基础与应用

Redis（Remote Dictionary Server）是一种基于 内存 的高性能键值对（Key-Value）数据库，常用于 缓存、分布式锁、消息队列、排行榜、计数器 等场景。

1. Redis 的特点

  • 高性能：数据存放在内存，读写速度极快（10w+ QPS）。
  • 数据结构丰富：支持 String、List、Hash、Set、ZSet 等。
  • 持久化：支持 RDB 快照和 AOF 日志，保证数据安全。
  • 支持集群：支持主从复制、哨兵模式、分片集群。
  • 多语言支持：几乎所有主流编程语言都支持 Redis 客户端。

2. Redis 常用数据结构

数据类型 存储内容 常见命令
String 字符串、数字 SET / GET / INCR
Hash 键值对集合（类似 Map） HSET / HGET / HGETALL
List 有序链表（消息队列常用） LPUSH / RPUSH / LPOP
Set 无序集合（去重、抽奖） SADD / SMEMBERS / SPOP
ZSet 有序集合（排行榜） ZADD / ZRANGE / ZREVRANK

3. Redis 持久化机制

  • RDB（快照）
    定期将内存中的数据保存到磁盘文件（默认 dump.rdb）。

    • 优点：适合大规模恢复，文件紧凑。
    • 缺点：可能丢失最后一次快照之后的数据。

  • AOF（日志）
    记录每一条写操作，默认 appendonly.aof。

    • 优点：数据更安全，最多丢失 1 秒数据。
    • 缺点：文件体积比 RDB 大，恢复速度慢。

生产环境常用 RDB + AOF 混合持久化。

4. Redis 高级应用场景

4.1 缓存

  • 用于热点数据存储，减少数据库压力。
  • 常见策略：
    • 缓存穿透：布隆过滤器、缓存空对象
    • 缓存击穿：热点 Key 设置互斥锁
    • 缓存雪崩：过期时间加随机值、分布式限流

4.2 分布式锁

使用 SETNX key value EX 30 实现分布式锁，并通过 Lua 脚本保证释放锁的原子性。

4.3 消息队列

利用 List 或 Stream 实现消息队列。

  • LPUSH + BRPOP → 简单队列
  • XADD + XREAD → 高级队列

4.4 计数器 / 排行榜

  • INCR key 实现计数器
  • ZADD 实现排行榜（分数为权重）

5. Redis 集群架构

主从复制

  • 主库负责写，从库负责读，提高性能。
  • 常用于读写分离、数据冗余。

哨兵模式

  • 监控主从状态，自动故障转移。

分片集群

  • 数据按槽（slot）分布到多个节点。
  • 解决单机内存瓶颈。
